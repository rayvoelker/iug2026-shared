# Amazon Business EDI Integration — Opening Session

## Presenters
- **Holbrook Sample** — CTLO, Public Library of Cincinnati and Hamilton County (CHPL)
- **Moses Lai** — Sr. Technical Product Manager, Amazon

## The Integration
- **Amazon Business EDI ordering** for Sierra libraries
- Order books from Amazon, download **brief MARC records** with order info, load into ILS
- Amazon's "grid" technology lets libraries add structured metadata (fund codes, locations, processing instructions) to orders before they flow into EDI
- **Sierra acquisitions API** is being developed on the III side to support this

## Key Points
- Amazon has had procurement tools but has been **focused on libraries for less than 1 year**
- MARC record quality has been improved; more features coming
- **EDI was the biggest request** from libraries — operational improvements over credit card purchasing
- Positions Amazon as a **primary materials vendor** (competing with B&T), not just a la carte
- **Patron-driven acquisitions** mentioned as a future possibility
- Amazon looking for **additional partnerships**

## CHPL Implementation
- Validated features with librarians throughout development
- Involved acquisitions, cataloging, processing, and fiscal office from early meetings
- **Launched mid-April 2026**
- Selectors still use Library Hub for pre-pub work
- Currently working out fund mapping between Amazon and ILS

## Implementation Advice
- Know up front: how to route to different locations, define a short list of goals
- Get solid MARC records up front to focus on change control
- Define all important features before implementation
- Align vendor tools to your workflows, not the other way around
- The more methodical the work, the better — EDI helps enforce that structure
