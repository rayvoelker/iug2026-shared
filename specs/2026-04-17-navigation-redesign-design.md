# Navigation & Usability Redesign

**Date:** 2026-04-17
**Status:** Approved
**Scope:** iug2026-shared repository

## Problem

The site looks polished but sacrifices usability. Links don't look obviously clickable (text links have no underline, nav items are low-contrast, card hover states are too subtle). There's no clear "show me all sessions" view — session pages are buried in the Deep Dives list on the homepage or require navigating through day pages. AI agents have no structured way to navigate the site.

## Goals

1. **Link affordance** — make all interactive elements unmistakably clickable across the site
2. **Session discoverability** — make the homepage a clear hub for finding any session page
3. **A/B test** — build two homepage variants so the user can compare and pick
4. **Agent navigation** — add `llms.txt` and `llms-full.txt` for AI agent discoverability

## Non-Goals

- Redesigning the visual brand (colors, card style, banner/footer stay as-is)
- Changing page templates (session, day, page, speakers remain unchanged)
- Adding search, sidebar nav, or other structural navigation patterns

## 1. CSS Link Affordance Overhaul

All changes in `static/style.css`.

### Text links (inside `.container`)

- Default state: `text-decoration: underline` (currently `none`)
- Hover: darken color, keep underline
- Keep teal base color but ensure sufficient contrast

### Nav bar

- Increase font size from `0.85rem` to `0.9rem`
- Active link: increase background opacity and/or add gold bottom-border accent
- Hover: more visible background change
- Overall: make it read as an obvious navigation menu

### Card links (`.day-card`, clickable `.card` elements)

- Add left-border color accent on hover
- More pronounced shadow lift on hover
- Explicit `cursor: pointer`

### Footer links

- Always underlined (small text needs obvious affordance)

## 2. Homepage Variant A — "Sessions-First"

Page structure top to bottom:

1. **Title + subtitle + info bar** — same as current
2. **Stats row** — same as current
3. **Session Notes by Day** — hero section:
   - Sessions grouped under day headings (Sunday, Monday, Tuesday, Wednesday)
   - Each session rendered as a card with title and description
   - This replaces both the current day grid AND deep dives list
4. **Guides** — separate section for standalone guides not tied to a day (Cloudflare, SSO Guide, Suggest-a-Purchase)
5. **Speaker Cards** — prominent link/card
6. **Conference Tracks** — reference section
7. **Post-COVID Attendance** — reference table

Template: `templates/index.html` (replaces existing)
Content: `content/index.md` (frontmatter restructured — sessions grouped by day, guides listed separately)

## 3. Homepage Variant B — "Hub"

Same top section, different main content:

1. **Title + subtitle + info bar** — same
2. **Stats row** — same
3. **Expanded Day Cards** — each day card shows its sessions inline:
   - Day card remains the visual anchor (date badge + day title)
   - Nested session links below each day (compact: title only, with optional track tag)
   - Clicking the day title goes to the day page; clicking a session goes to the session page
4. **Guides** — same as variant A
5. **Speaker Cards** — same
6. **Tracks + Attendance** — bottom

Template: `templates/index-b.html` (new)
Content: `content/index-b.md` (temporary, removed after A/B decision)

## 4. Agent Navigation — llms.txt

### `docs/llms.txt`

Auto-generated by `build.py`. Contains:
- Site title and description
- List of all pages organized by category (sessions, day overviews, guides, special pages)
- Each entry: URL, title, one-line description
- Pointer to `speakers.json` for structured speaker data

### `docs/llms-full.txt`

Extended version with full rendered content of every page in a single document, separated by clear page boundaries. Allows agents to ingest the whole site in one request.

### Build integration

- New `generate_llms_txt(site, pages)` function in `build.py`
- Runs after pages are built, before static copy
- Uses the same `pages` metadata already available from `discover_content()`

### Discovery

- Add `<link rel="help" href="llms.txt">` to `templates/base.html`

## 5. A/B Comparison Workflow

1. Build both variants — `index.html` (variant A) and `index-b.html` (variant B)
2. User views both on the dev server
3. User picks one (or combines elements from both)
4. Remove the losing variant's template and content file
5. Final commit with the chosen homepage
